Day 4 — 質問分岐エンジンの構築

5. 概要と追加機能

従来の線形出題（order昇順で全問出題）を拡張し、前の回答に応じた動的な質問選択を実現する分岐エンジンを構築した。

機能説明
必須/任意フラグ質問ごとに required を設定。任意質問は低スコアでも面接を fail にしない
カテゴリ分類質問を「技術」「コミュニケーション」等のカテゴリで分類可能
分岐ルールJSON形式で条件付き出題ルールを定義。条件不一致の質問を自動スキップ

6. DB変更（マイグレーション）

db/migrate/20260306120000_add_branching_to_questions.rb

カラム型デフォルト説明
requiredbooleantrue必須質問フラグ
categorystringnull質問カテゴリ
branching_rulesjsonnull分岐条件（JSON）
後方互換: 既存の質問は required=true, branching_rules=null となり、従来通り無条件で出題される。

7. 分岐ルール設計と評価フロー

branching_rules のJSON構造

// 例: 質問1で「Python」を選択した場合のみ出題 { "conditions": [ { "source_question_order": 1, "type": "selected_option", "value": "Python", "action": "include" } ], "default_action": "skip" }

条件タイプとアクション

type説明判定方法
selected_optionMCQ選択肢一致回答テキスト == value（大文字小文字無視）
score_aboveスコアが指定値以上final_score >= value
score_belowスコアが指定値未満final_score < value
answered回答済みかどうかInterviewResponse の存在確認
action説明
include条件一致時に出題する
skip条件一致時にスキップする

9. 使用例と後方互換性

使用例

例: MCQ回答による質問分岐

後方互換性

項目既存データへの影響
branching_rules = null従来通り無条件で出題（挙動変更なし）
required = true（デフォルト）既存質問は全て必須扱い（挙動変更なし）
category = null動作に影響なし（表示・分類用途のみ）
APIレスポンスrequired フィールドが追加されるのみ
既存の面接フローは一切影響を受けません。 新カラムは全てデフォルト値が設定されており、マイグレーション適用のみで既存データとの互換性が維持されます。

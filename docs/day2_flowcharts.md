## 1. 面接フロー全体図

### 1.1 システム全体アーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│                      ブラウザ (User)                         │
│                                                             │
│  Step1: 面接選択    Step2: 面接実施    Step3: 結果確認       │
│  ┌──────────┐      ┌──────────┐      ┌──────────┐          │
│  │Situation │─────▶│ Q&A Loop │─────▶│ Results  │          │
│  │ 選択     │      │ 質問応答 │      │ 表示     │          │
│  └──────────┘      └──────────┘      └──────────┘          │
└───────┬─────────────────┬──────────────────┬────────────────┘
        │                 │                  │
   POST /start    POST /submit_answer  POST /complete
   GET /next_question    (繰り返し)    GET /status
        │                 │                  │
┌───────▼─────────────────▼──────────────────▼────────────────┐
│                    Rails API サーバー                        │

### 3.2 回答送信〜次質問の内部フロー

```
[回答を送信] クリック
      │
      ▼
┌─ submitAnswer() ─────────────────────────────┐
│                                               │
│  1. isSubmitting ガードチェック               │
│  2. 入力バリデーション                        │
│     ├─ テキスト入力あり?                     │
│     ├─ 音声ファイルあり?                     │
│     ├─ 動画ファイルあり?                     │
│     ├─ ブラウザ録音あり?                     │
│     └─ MCQ選択あり?                          │
│     → いずれもなし → alert                    │
│                                               │
│  3. FormData 構築                             │
│     ├─ question_id                            │
│     ├─ text_answer (テキスト)                │
│     ├─ audio_file (ファイル or 録音blob)     │
│     ├─ video_file (動画)                     │
│     └─ selected_option (MCQ選択)             │
│                                               │
│  4. POST /api/interviews/:id/submit_answer   │
│     └─ ボタン: "送信中..." (disabled)        │
│                                               │
│  5. レスポンス処理                            │
│     ├─ success: false → alert                │
│     └─ success: true                          │
│        ├─ フォームリセット                   │
│        ├─ ステータス: "次の質問を読み込み中" │
│        └─ loadNextQuestion() 呼び出し        │
│                                               │
│  6. finally                                   │
│     ├─ isSubmitting = false                   │
│     ├─ ボタン有効化                          │
│     └─ テキスト: "回答を送信"                │
└──────────────────┬────────────────────────────┘
                   │
                   ▼
┌─ loadNextQuestion() ─────────────────────────┐
│                                               │
│  1. GET /api/interviews/:id/next_question    │
│                                               │
│  2. レスポンス分岐                            │
│     │                                         │
│     ├─ interview_complete: true              │

### 4.3 スコア計算の詳細

```
┌──────────────────────────────────────────────────────────┐
│                   評価スコア計算式                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ■ 個別回答スコア (ResponseEvaluator)                   │
│                                                          │
│  final_score = (relevance_score × 0.40)                 │
│              + (correctness_score × 0.40)                │
│              + (clarity_score × 0.20)                    │
│                                                          │
│  各スコア範囲: 0〜100                                   │
│  final_score 上限: 100                                   │
│  合格閾値: PASS_THRESHOLD = 70                          │
│                                                          │
│  ■ 重み配分の意味                                       │
│  ┌────────────┬────────┬──────────────────────────┐     │
│  │ 評価軸     │ 重み   │ 評価内容                 │     │
│  ├────────────┼────────┼──────────────────────────┤     │
│  │ Relevance  │ 40%    │ 質問に対する関連度       │     │
│  │ Correctness│ 40%    │ 情報の正確性             │     │
│  │ Clarity    │ 20%    │ 回答の明確さ・構成力     │     │
│  └────────────┴────────┴──────────────────────────┘     │
│                                                          │
│  ■ 面接全体スコア (SessionManager)                      │
│                                                          │
│  average_score = Σ(evaluated_responses.final_score)     │
│                  ÷ evaluated_responses.count              │
│                                                          │
│  合格判定: average_score >= 70 → PASSED                 │
│            average_score <  70 → FAILED                  │
│                                                          │
│  ■ MCQ（選択式）スコア                                  │
│  正解 → relevance=100, correctness=100, clarity=100     │
│  不正解 → relevance=0, correctness=0, clarity=0         │
│                                                          │
│  ■ スコア例                                             │
│  ┌────┬──────────┬────────────┬─────────┬───────┬──────┐│
│  │ Q# │Relevance │Correctness │Clarity  │Final  │Pass? ││
│  ├────┼──────────┼────────────┼─────────┼───────┼──────┤│
│  │ 1  │ 85       │ 80         │ 90      │ 84.0  │ ✅   ││
│  │ 2  │ 70       │ 65         │ 75      │ 69.0  │ ❌   ││
│  │ 3  │ 90       │ 95         │ 85      │ 91.0  │ ✅   ││
│  ├────┼──────────┼────────────┼─────────┼───────┼──────┤│
│  │平均│          │            │         │ 81.3  │ ✅   ││
│  └────┴──────────┴────────────┴─────────┴───────┴──────┘│
│                                                          │
│  → 平均81.3 >= 70 → final_status: PASSED              │
└──────────────────────────────────────────────────────────┘
```

### 4.4 ステータス遷移図

```
■ Interview.status

  not_started(0) ──start!──▶ in_progress(1)
                                │
                     ┌──────────┼──────────┐
                     │          │          │
                  complete!   fail!    abandon
                     │          │          │
                     ▼          ▼          ▼
              completed(2)  failed(3)  abandoned(4)


■ InterviewResponse.evaluation_status

  pending(0) ──evaluate開始──▶ evaluating(1)
                                    │
                            ┌───────┴───────┐
                            │               │
                         成功             失敗
                            │               │
                            ▼               ▼
                      completed(2)      failed(3)


■ InterviewResult.final_status

  ┌─────────────────────────────────────────┐
  │  complete API 呼び出し時に決定          │
  │                                         │
  │  average_score >= 70 → passed(0) 合格  │
  │  average_score <  70 → failed(1) 不合格│
  │  (incomplete(2) は未使用)               │
  └─────────────────────────────────────────┘
```

### 4.5 即時リジェクト（現在の実装）

```
┌──────────────────────────────────────────────────────────┐
│  現在の即時リジェクト実装                                │
│  (ResponseEvaluator#check_interview_continuation)        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  個別回答の evaluation 完了後:                           │
│                                                          │
│  score < PASS_THRESHOLD(70) の場合                      │
│    → SessionManager.fail_interview(interview, reason)   │
│    → Interview.status = :failed                          │
│    → ended_at = 現在時刻                                │
│                                                          │
│  ⚠️ 注意: 現在は1問でもfailすると面接全体がfailになる   │
│  → 改善案: 連続N問失敗時のみfail、または全問終了後判定  │
│                                                          │
│  将来の改善案:                                           │
│  ┌────────────────────────────────────────────────┐     │
│  │ Option A: 全問終了後に一括判定（推奨）         │     │
│  │  → 個別failでは面接を止めない                  │     │
│  │  → complete時にaverage_scoreで最終判定         │     │
│  │                                                │     │
│  │ Option B: 連続失敗での打ち切り                 │     │
│  │  → 3問連続 score < 70 で面接を打ち切り        │     │
│  │  → final_status: failed                        │     │
│  │                                                │     │
│  │ Option C: Situationごとに設定可能              │     │
│  │  → situations.min_score_required カラム追加    │     │
│  │  → situations.fail_policy カラム追加           │     │
│  │    ('per_question' | 'average' | 'consecutive')│     │
│  └────────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────────┘
```

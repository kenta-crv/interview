4. RSpecテスト修正（18件 → 0件の失敗）

4.1 原因1: Factory traitのステータス遷移バリデーション違反（13件）

Interviewモデルの valid_status_transition バリデーションにより、not_started → completed 等の直接遷移が禁止されている。しかしFactoryの :completed, :abandoned, :failed トレイトは status { :completed } のようにステータスを直接設定していた。

修正前

trait :completed do status { :completed } started_at { 1.hour.ago } ended_at { Time.current } end

修正後

trait :completed do after(:create) do |interview| interview.start! interview.complete! end end

同様に :in_progress, :failed, :abandoned, :timed_out トレイトも after(:create) コールバック方式に修正。正しいステータス遷移（not_started → in_progress → completed/failed/abandoned）を経由するようにした。

4.2 原因2: APIテストの認証設定不備（2件）

テスト問題修正
POST /startlet(:user) が遅延評価でユーザー未作成let!(:user) に変更
rejects invalid tokenテストモードONで認証バイパス発生テストモードを無効化、Devise応答を許容
